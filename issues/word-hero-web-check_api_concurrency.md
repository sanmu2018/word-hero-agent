# Issue: 检查接口并发

## 背景
`word-hero-web` 需要验证前端在并发请求和状态更新下不会对后端接口造成过载，也能正确合并响应与 UI 状态，确保多标签或快速操作场景稳定。

## 任务
- 检查 `src/services/*` 中封装的 API 调用，确认在重复点击、并发请求、重试政策下不会创建竞态条件（例如重复标记、冲突的状态写入）。
- 针对关键交互（单词标记、搜索、分页加载等）编写 E2E 或集成测试，模拟高并发请求并观察 UI 状态是否一致。
- 与后端协调压测，确保前端的请求节流/防抖策略可控，必要时增加队列或状态检查。

## 验收标准
- 输出测试计划与结果，附带脚本、自动化用例或复现步骤。
- 记录发现的问题与修复方案（含 PR/Commit 链接）；若未发现问题，也需描述验证方法。
- 在此 Issue 中持续更新进展，并在完成后标记完成同时 @word-hero-agent。
