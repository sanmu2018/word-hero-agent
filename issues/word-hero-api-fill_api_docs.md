# Issue: 补全 API 文档

## 背景
`word-hero-api` 服务新近初始化，当前仓库缺少与线上后端 (`word-hero`) 一致的接口文档。产品与下游（web、agent）团队需要一份可信的 API 参考，以便协同开发与排查问题。

## 任务
- 从 `word-hero` 仓库读取最新接口（推荐在 `.tmp/repos/word-hero` 克隆只读副本），梳理所有面向外部的 REST 入口，至少覆盖单词检索、标记、用户管理等模块。
- 为每个接口补齐请求/响应结构、状态码、鉴权要求以及示例 payload，整理成统一文档（建议使用 `docs/api.md` 或 OpenAPI 描述）。
- 如果 `word-hero` 缺失可用于生成文档的注释/元数据，给出需要的补充清单或自动化脚本建议。
- 在 README 或仓库根文档中标注该 API 文档的生成与更新流程，确保其他服务知道如何同步最新协议。

## 验收标准
- 仓库新增或更新的 API 文档可在 `word-hero-api` 中直接查看，并包含上述模块的详细条目信息。
- Issue 评论中附上相应的 PR/commit 链接，以及文档存放路径和生成流程说明。
- 如果需要其他服务配合，请在文档中明确依赖及后续动作。

> 完成后请在此 Issue 更新进度，并 @word-hero-agent 以确认交付。
