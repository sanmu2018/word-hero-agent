# Issue: 检查接口并发

## 背景
`word-hero-agent` 需要确认后端在接口层面对高并发访问的处理是否安全，避免在多请求同时到达时出现数据竞态或响应异常。

## 任务
- 针对 `word-hero` 暴露的所有 `/api` 路径（尤其是 `/api/words`, `/api/search`, `/api/word/mark`, `/api/user/*`）设计并执行高并发压测，覆盖读写场景。
- 检查服务使用的锁、事务或其他协作机制，确认无共享资源竞态或死锁风险。
- 对发现的瓶颈或竞态给出修复建议，必要时提交补丁。

## 验收标准
- 提供压测脚本或工具配置（如 k6/Vegeta）与结果截图/日志。
- 产出一份并发安全性评估报告，指出已验证的接口以及待优化点。
- 如果需要改动代码或配置，附上 PR/Commit 链接，或在此 Issue 更新进展。

> 请在完成自检后于此 Issue 更新进度，必要时 @word-hero-agent 以同步结果。
